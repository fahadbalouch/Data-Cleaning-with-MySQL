# Data-Cleaning-with-MySQL
This project focuses on the critical data cleaning phase of the data analytics pipeline. Using a dataset of global company layoffs, I demonstrated how to transform raw, messy data into a clean, structured format suitable for professional Exploratory Data Analysis (EDA) and visualization. This project follows best practices, including maintaining data lineage by creating staging tables to ensure raw data integrity.
# Key Objectives
Data Integrity: Establishing a "staging" environment to prevent accidental modification of source data.
Standardization: Resolving inconsistencies in data entries to allow for accurate grouping and aggregation.
Data Quality: Handling null values, missing records, and duplicate entries systematically.
Data Type Optimization: Converting unstructured text data into formal SQL data types for reliable time-series analysis.
# Skills
MySQL: Database management, schema creation, data import, complex SQL queries.
Data Wrangling: Normalization, data cleaning, and preprocessing.
Window Functions: ROW_NUMBER()
Table Manipulation: INSERT INTO, UPDATE, ALTER TABLE, JOIN (self-joins).
# Conclusion
This project results in a high-quality, "analysis-ready" dataset. By following this workflow, you ensure that downstream visualizations and trends are based on accurate and reliable data. This is a foundational project for any data analyst's portfolio, showcasing the ability to handle the "real-world" messiness of raw data.
